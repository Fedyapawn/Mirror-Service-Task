Mirror Generic Network Message Service
Тестовое задание на позицию C# Unity Developer.
Описание проекта
Реализован сервис обмена сетевыми сообщениями между сервером и клиентом на базе плагина Mirror.
Решенная проблема:
Стандартный механизм Mirror отключает клиента, если на его стороне не зарегистрирован обработчик для конкретного типа сообщения. В данном решении эта проблема обходится без модификации исходного кода плагина.
Как это работает (Архитектура):
Generic Wrapper: Используется универсальное сообщение-оболочка GenericNetworkMessage, содержащее имя типа и массив байт (payload).
Единая точка входа: В Mirror регистрируется только один системный обработчик для этой оболочки. Это гарантирует, что клиент никогда не будет отключен из-за «неизвестного» типа сообщения.
Система подписок: Реализован сервис подписок на стороне сервера. Сервер отправляет данные только тем клиентам, которые явно прислали запрос на подписку (SubscribeRequestMessage).
Сериализация: Упаковка и распаковка пользовательских структур в байты происходит вручную через NetworkWriter/NetworkReader, что позволяет динамически обрабатывать любые типы данных, реализующие NetworkMessage.
Технологический стек
Unity: 2022.3.62f3
Mirror: Сетевой движок
VContainer: DI-контейнер для управления зависимостями
KCP Transport: Стандартный транспорт для передачи данных
Инструкция по запуску
Склонируйте репозиторий.
Откройте проект в Unity версии 2022.3.62f3.
Откройте сцену Assets/_Project/Scenes/Main.unity.
Для тестирования взаимодействия:
Соберите билд (File -> Build and Run) или используйте второе окно редактора.
В одном окне запустите Host.
В другом окне запустите Client.
Сценарий проверки:
На Клиенте нажмите кнопку Subscribe. В консоли появится подтверждение запроса.
На Сервере (Хосте) нажмите кнопку Send Hello.
Результат: В консоли Клиента отобразится текст: [Client] Received: Hello Client!.
Структура кода
Scripts/Core/NetworkMessages.cs — Сетевые структуры и оболочки.
Scripts/Networking/NetworkService.cs — Ядро системы: логика сериализации, регистрации обработчиков и рассылки сообщений.
Scripts/DI/GameLifetimeScope.cs — Настройка зависимостей и инициализация сервиса.
Scripts/Logic/NetworkTestController.cs — Компонент для тестирования сценария (кнопки и логи).
